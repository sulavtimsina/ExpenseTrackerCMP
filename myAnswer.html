Project Structure and Main Components

This project is a modern, cross-platform Expense Tracker application built with Kotlin Multiplatform and Compose Multiplatform. It supports Android, iOS, and desktop platforms, sharing business logic and UI code while allowing platform-specific integration.

High-Level Overview

  • Shared Module (composeApp/): Core business logic, UI components, data layer, and resources shared across platforms.
  • iOS App (iosApp/): Native iOS application using SwiftUI and the generated Compose framework.
  • Supabase (supabase/): Backend configuration and database migrations for local development with Supabase.
  • Scripts (scripts/ & setup.sh): Utility scripts to configure the Android build environment and set up the Supabase local environment.
  • Build Configuration: Gradle wrapper, version catalogs, and root-level Gradle scripts (build.gradle.kts, settings.gradle.kts).
  • Documentation & Assets: README.md, CONTRIBUTING.md, LICENSE, sample screenshots (screenshots/).

Detailed Directory Layout

composeApp/ (Shared Multiplatform Module)

  • src/commonMain: Shared code including:
    • Business logic and domain (com/sulavtimsina/expensetracker): analytics/, expense/, settings/, core/, data/, di/.
    • SQLDelight schema (sqldelight/) for type-safe database operations.
    • Shared UI components with Compose.
  • src/androidMain: Android-specific code and resource adapters.
  • src/iosMain: iOS-specific integration for the Compose framework.
  • src/desktopMain: Desktop entry point and app launcher.
  • build.gradle.kts: Module build configuration.

iosApp/ (iOS Application)

  • iosApp.xcodeproj: Xcode project file.
  • iosApp/iosApp: SwiftUI-based UI (ContentView.swift, iOSApp.swift) integrating ComposeApp.framework.
  • Configuration/Config.xcconfig: Build settings for Xcode.

supabase/ (Backend Configuration)

  • supabase/config.toml: Local Supabase project settings.
  • supabase/migrations/: Database migration scripts for setting up tables.
  • supabase.env.example: Environment variable template for connecting to Supabase.

scripts/ & setup.sh

  • setup.sh: Automates Android SDK detection, installation of command-line tools, and generation of local.properties for Android builds.
  • scripts/setup-supabase.sh & setup-supabase-simple.sh: Scripts to initialize and run Supabase locally.

Root-Level Configuration

  • build.gradle.kts: Top-level Gradle build script defining plugins and common tasks.
  • settings.gradle.kts: Includes modules (composeApp, iosApp).
  • gradle/: Version catalog libs.versions.toml for dependency management.
  • gradlew & gradlew.bat: Gradle wrapper scripts.
  • gradle.properties & local.properties: Global and local settings (Android SDK path).

Supporting Files
